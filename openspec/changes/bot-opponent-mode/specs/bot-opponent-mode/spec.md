## ADDED Requirements

### Requirement: Bot opponent match setup
The system SHALL support a local solo match mode in which one human player competes against one AI-controlled opponent. The AI opponent SHALL occupy the non-human player slot for the match.

#### Scenario: Start bot match
- **WHEN** the player starts a solo bot match
- **THEN** the match initializes with one human-controlled player and one AI-controlled opponent

#### Scenario: Opponent is AI controlled
- **WHEN** the match begins
- **THEN** the opponent's actions are generated by the bot rather than direct player input

### Requirement: Bot action generation
The system SHALL generate opponent actions for each tick based on the current game state and available resources. The bot SHALL only queue actions that pass normal game validation.

#### Scenario: Bot queues legal actions
- **WHEN** the bot has resources and the current state allows a valid action
- **THEN** the bot queues at least one legal action before the tick resolves

#### Scenario: Bot skips invalid actions
- **WHEN** the bot would select an action that fails validation
- **THEN** the action is not applied and the tick continues with the remaining valid actions

### Requirement: Bot decisions are deterministic
Given the same game state and configuration, the bot opponent SHALL produce the same actions.

#### Scenario: Identical state yields same move
- **WHEN** two matches reach the same game state with the same configuration
- **THEN** the bot chooses the same actions in both matches
